# AIタスクマネジメントツール「SEBASTIAN」を開発中です

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2019.51.59.png)

日々の仕事をしていると、タスクは綺麗な形では降ってきません。

メール、チャット、会議、口頭依頼、ふと思いついた改善案、後でやろうと思った小さな作業。
それらは常に断片的で、あとから整理しようと思っても、気づけば記憶の中に埋もれてしまいます。

そこで、雑多な業務メモをAIで整理し、日報・タスク・週報へ変換するためのデスクトップアプリとして、**SEBASTIAN** を開発しています。

## SEBASTIAN(セバスチャン)とは

SEBASTIANは、ローカル環境で動作するAIタスクマネジメントツールです。

Github: https://github.com/roman-ease/Sebastian

一般的なタスク管理ツールのように、最初から綺麗なタスク名、期限、優先度、カテゴリを入力するのではなく、
まずは思いついたことや作業メモを雑に書き込みます。

その後、AIがメモの内容を読み取り、以下のような形に整理します。

- タスク候補の抽出
- 日報案の生成
- 週報案の生成
- 作業ログの整理
- スケジュール候補の提示

ユーザーは、AIの提案内容を確認し、必要に応じて採用・修正します。

SEBASTIANの基本方針は、
**「AIが勝手に確定する」のではなく、「AIが整理して、人間が決める」**
というものです。

## 開発の背景

業務の中では、明確なタスクとして登録する前の情報が多く発生します。

例えば、以下のようなものです。

- 会議中に出た小さな宿題
- チャットで軽く依頼された対応
- 後で調べようと思った技術メモ
- 日報に書くほどではないが、記録しておきたい作業
- 週報作成時に思い出せなくなる細かい対応
- 進行中の作業に紐づく判断や検討内容

こうした情報は、発生した瞬間に完璧なタスクとして整理するのが難しいです。

一方で、放置すると日報や週報を書くときに思い出せなくなります。
また、あとから「なぜこの作業をしたのか」「いつ対応したのか」を振り返りにくくなります。

そこで、まずは雑に記録し、あとからAIで整理する仕組みが必要だと考えました。

## コンセプト

SEBASTIANのコンセプトは、
**「雑に書いた業務メモを、AIが整理して差し出してくれる執事」**
です。

名前のSEBASTIANも、執事のように控えめに補佐する存在をイメージしています。

ユーザーの代わりに勝手に仕事を進めるのではなく、
ユーザーの手元に散らばった情報を集めて、
「本日の作業はこちらです」
「未完了のタスクはこちらです」
「週報に使えそうな内容はこちらです」
と整理して提示することを目指しています。

## 想定している使い方

基本的な利用の流れは以下の通りです。

1. 作業中に思いついたことをメモする
2. その日の作業内容を雑に記録する
3. AIがメモからタスク候補を抽出する
4. 必要なものだけタスクとして登録する
5. 1日の終わりに日報案を生成する
6. 週末に週報案を生成する

重要なのは、最初の入力をできるだけ軽くすることです。

「これはタスクなのか」
「期限はいつなのか」
「優先度はどれくらいなのか」

といった整理は、最初から完璧にやる必要はありません。
まず書き残しておき、あとからAIと一緒に整えます。

## 主な機能

現時点で想定している主な機能は以下です。

### 今日のメモ

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2019.53.11.png)

その日に発生した作業、依頼、気づき、調査メモなどをまとめて記録する機能です。

タスク管理ツールに登録するほどではない内容も、まずはここに書き込みます。

### AIによるタスク抽出

今日のメモをAIが読み取り、タスク候補を抽出します。

例えば、以下のようなメモがあった場合、

> 田中さんからDNS移行手順の確認依頼。週内にチェックして返す。

AIは次のようなタスク候補を提示します。

> DNS移行手順を確認し、田中さんへ回答する

この段階では、まだ確定タスクではありません。
ユーザーが内容を確認し、必要に応じて採用します。

### 日報生成

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2019.53.27.png)

今日のメモや完了したタスクをもとに、日報案を生成します。

日報には、以下のような内容を含める想定です。

- 本日の作業内容
- 完了したタスク
- 進行中のタスク
- 課題・懸念点
- 明日以降の予定

出力形式はMarkdownを想定しています。

```text
Nippo_yyyymmdd.md
```

### 週報生成

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2019.53.29.png)

日報やタスクログをもとに、週報案を生成します。

日々の作業ログを蓄積しておけば、週報作成時に記憶を掘り起こす必要が減ります。

```text
Shuho_yyyymmdd.md
```

### タスク管理

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2019.53.18.png)

AIが抽出したタスクや、手動で登録したタスクを管理します。

想定しているステータスは以下です。

* 未着手
* 進行中
* 完了
* 保留
* アーカイブ

タスクには、作業ログや変更履歴も紐づける予定です。

### 週スケジュール

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2019.53.23.png)

タスクや予定を週単位で確認するための画面です。

カレンダーアプリほど大掛かりなものではなく、
「今週やること」を見通すための簡易的なスケジュール管理を想定しています。

## 一連のデモ

まず、1日のメモを取ります。綺麗に書けばそれだけ精度は出ますが雑で構いません。

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2020.02.09.png)

雑なメモが取れたら画面下部にある「日報を作成する」ボタン、もしくは日報画面を展開します。日報画面が出たら 「日報案を生成する」をクリック。すると、羽ペンが動くアニメーションとともにバックグラウンドでAPIにメモの内容を飛ばし、AIに内容を精査してもらいます。

![](public/uploads/2026/05/スクリーンショット%202026-05-07%2019.53.27.png)

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2020.02.33.png)

羽ペンのアニメーションが終わると、AIによって成型された日報が表示されます。今回はOllamaで生成してもらってます。

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2020.02.54.png)

内容を確認し誤りのある部分等をユーザー自身でチェックし、その場でMarkdownの日報を修正。修正が完了すれば「承認・保存する」ボタンをクリック。

すると、清書された日報が表示されるので、画面下部にある「タスク候補を抽出する」をクリック。日報と同様にAIによってタスクになりそうな内容を精査します。

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2020.03.04.png)

AIによるタスク精査が完了すると以下のようにタスク候補がいくつか出力されるので、内容を確認し、チェックボックスにチェックを入れて「タスクに追加」をクリック。

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2020.03.52.png)

追加のボタンを押したタスクらは左側サイドパネルの「タスク」に格納される。メモの内容に期限が記載されていれば自動で期限も入れた状態でタスク化してくれますが、必要に応じてそうした情報は追記してください。

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2020.04.04.png)

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2020.16.59.png)

期限が設定されれば週スケジュールにタスクが追加され、タスクをクリックすることでその詳細を確認することができます。

![](/uploads/2026/05/%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%BC%E3%83%B3%E3%82%B7%E3%83%A7%E3%83%83%E3%83%88%202026-05-07%2020.18.45.png)

このような流れで1日のメモのみを残し、残りのタスクのカード化や期限設定などタスクマネジメントの面倒な部分を極力AIに任せる。

これにより、小さな面倒を削減し、チリつもで作業効率を上げる。これがSebastianの目的です。

## 技術構成

SEBASTIANは、Windowsデスクトップアプリとして開発しています。

主な技術構成は以下です。

| 項目 | 内容 |
| --- | --- |
| アプリ基盤 | Tauri |
| フロントエンド | React |
| 言語 | TypeScript / Rust |
| データベース | SQLite |
| スタイリング | Tailwind CSS |
| 保存方針 | ローカルファースト |
| 出力形式 | Markdown |

クラウドサービスとして作るのではなく、まずはローカルで完結する個人用ツールとして設計しています。

業務メモや日報には、個人的な情報や社内向けの情報が含まれる可能性があります。
そのため、最初からクラウド同期を前提にするのではなく、ローカル環境で安全に扱える構成を優先しています。

## デザイン方針

SEBASTIANは、単なる業務ツールではなく、
「執事が仕事を整理してくれる」ような世界観を持たせています。

デザインの方向性は、クラシカルな書斎や羊皮紙、セピア調の雰囲気です。

想定しているテーマは以下です。

| テーマ | イメージ |
| --- | --- |
| Classic | 羊皮紙と深いネイビー |
| Dark | 深夜の書斎と真鍮ランプ |
| Sepia | ビクトリア朝風のウォールナット家具 |

ただし、キャラクター性を強く出しすぎると実務利用の邪魔になります。
そのため、あくまで控えめに、実用性を損なわない範囲で世界観を取り入れる方針です。

## SEBASTIANで重視していること

SEBASTIANで最も重視しているのは、入力のハードルを下げることです。

タスク管理は、続かなければ意味がありません。

最初から細かい項目を入力させると、忙しいときほど記録が後回しになります。
そして、後回しになった記録は、そのまま失われます。

SEBASTIANでは、最初の入力をできるだけ雑でよいものにします。

* とりあえず書く
* あとから整理する
* AIに候補を出させる
* 最後は自分で決める

この流れにすることで、日々の業務記録を継続しやすくすることを目指しています。

## 目指しているもの

SEBASTIANは、完璧なプロジェクト管理ツールを目指しているわけではありません。

Backlog、Jira、Notion、Todoistのような本格的な管理ツールとは目的が異なります。

SEBASTIANが目指しているのは、
**個人の頭の中に散らばった仕事の断片を、あとから扱える形に整えること**
です。

特に、以下のような人に向いているツールになると考えています。

* 突発対応が多い人
* 日報や週報を書く必要がある人
* チャットや口頭依頼が多い人
* 細かい作業をあとから思い出せなくなる人
* タスク管理ツールへの入力が続かない人
* AIを業務補助に使いたいが、完全自動化には不安がある人

## まとめ

SEBASTIANは、雑多な業務メモをAIで整理し、日報・タスク・週報へ変換するためのローカルデスクトップアプリです。

コンセプトは、
**「雑に書いた仕事の断片を、執事のように整理して差し出すAIタスクマネジメントツール」**
です。

タスク管理をきれいに始めるのではなく、
雑に記録して、あとから整える。

そのための個人用業務補佐ツールとして、SEBASTIANを開発しています。
